
⸻

■ Selenium（無料・オープンソース）

Seleniumは、最も広く利用されているWebブラウザ自動操作ツールで、クリック、入力、遷移、検証、スクリーンショットなどを自動化できます。Java、Python、JavaScriptなどの言語と、Chrome、Firefox、Edgeなどの主要ブラウザに対応しています。

PC環境だけでなく、スマホやタブレットの**ブラウザをエミュレート（user agent切替や画面サイズ指定）**することで、レスポンシブ対応テストも可能です。

ローカル環境での実行は容易で、Selenium GridやCI/CDと組み合わせればAWS上でも分散実行が可能です。

AWSにデプロイする場合の構成：
	•	EC2：テストスクリプト＋ヘッドレスブラウザ実行環境
	•	VPC／セキュリティグループ：対象Webサイトへの接続制御
	•	IAM（任意）：レポートをS3へアップロードする際の権限設定
	•	S3（任意）：実行ログやスクリーンショットの保存

Device Farm：不要（非対応）
※ Seleniumはネイティブアプリの操作はできず、モバイルWebの操作のみに向いています。

⸻

■ Katalon Studio（無料／有償）

Katalonは、Web／API／モバイルアプリ／デスクトップアプリの自動テストに対応したツールです。Seleniumベースですが、GUI操作、レコーディング、データ駆動、テスト管理機能も備えています。

ローカル環境にインストールして使用するだけでなく、**スマホやタブレット実機（USB接続 or エミュレーター）**でのモバイルアプリテストも可能です。Appiumベースのモバイルテストスクリプトも生成可能です。

AWSでの利用：
CLI版をEC2上で実行したり、TestOps（Katalonのクラウド）からテストスケジューリングを行うことも可能です。

AWSで必要な構成：
	•	EC2：CLIスクリプトの実行環境
	•	IAM：TestOps連携、S3とのやり取りに必要
	•	S3（任意）：テストログやレポート保存用
	•	VPC／SG：対象サーバとのネットワーク制御
	•	（外部）TestOps：CI/CD・結果管理・チーム統合向け

Device Farm：必要に応じて使用可能
→ AppiumスクリプトをDevice Farmで実行する構成が可能。実機テストをクラウド化したい場合に便利。

⸻

■ TestComplete（有償）

TestCompleteは、エンタープライズ向けのGUI自動化ツールで、Windowsアプリ・Web・モバイルに対応しています。録画・再生・スクリプト化・スマートオブジェクト認識など高機能で、企業での大規模テストに向いています。

Windows専用アプリのため、AWS上ではWindows EC2にインストールする必要があります。
スマホやタブレットの操作には、基本的にエミュレータ連携や別途モバイルツールとの連携が必要になります。

AWSで必要な構成：
	•	Windows EC2：TestComplete / TestExecute実行用
	•	RDP：リモートでのGUI操作用
	•	VPC／SG：対象環境への疎通制御
	•	S3／IAM（任意）：CI/CD・ログ保存用

Device Farm：非対応
TestCompleteはAWS Device Farmとは統合できません。実機テストを行いたい場合は別ソリューションが必要です。

⸻

■ AWS Device Farm（実機テスト専用サービス）

AWS Device Farmは、iOS／Androidの本物のスマホやタブレットをクラウド上で借りて自動テストを実行できるサービスです。Appium／XCUITest／Espressoなどのスクリプトをアップロードして、自動的にテストが行われます。

KatalonやAppiumと組み合わせることで、ネイティブアプリの実機テストが可能になります。
UI操作の録画、ログ収集、エラー分析などもでき、端末を持っていなくてもテスト可能です。

⸻

【補足まとめ】
	•	PC・スマホ・タブレットのブラウザUIのみなら Selenium / Katalon で十分
	•	ネイティブアプリのテスト（スマホにインストールするApp）なら Device Farm が便利
	•	Windowsアプリなど特殊UIなら TestComplete（ただしスマホには不向き）
	•	AWSにツールをデプロイしたら、ローカルからSSH、API、CIツールなどで呼び出して実行可能

■ Selenium（無料）

① 主な機能
→ オープンソースのWebブラウザ自動操作ツール。
→ Chrome、Firefox、Edge等に対応し、入力・クリック・検証などを自動化可能。
→ 複数言語（Java、Python、JavaScriptなど）をサポートし、柔軟な拡張が可能。

② クライアントで実行できるか
→ 実行可能。ローカルPC上でテストスクリプトを作成・実行できる。
→ Selenium GridやDockerを用いた分散実行も可能。

③ サーバで実行する場合に必要なAWS環境
→ EC2インスタンスにテストスクリプトとヘッドレスブラウザ環境を構築して実行可能。
→ 複数ノードでの分散テストを行う場合、Selenium GridをEC2またはコンテナで構成可能。
→ CIツール（Jenkins、GitHub Actions等）と統合して運用するケースが多い。

備考：
	•	柔軟性が高い一方、スクリプト保守コストもある。
	•	エンジニア主体のテスト自動化チームに向いている。
	•	AWS Device Farmと連携すればマルチデバイス対応も可能（別料金）。

⸻

■ Katalon Studio（無料版あり / 有料プランあり）

① 主な機能
→ SeleniumベースのGUIテストツール。Web、API、モバイル、デスクトップに対応。
→ レコーディング、データ駆動テスト、アサーション設定、GUIでの操作が可能。

② クライアントで実行できるか
→ 実行可能。GUIアプリとしてローカルにインストールして使用。
→ CLIモードでスクリプトを出力し、サーバやCI/CDでの実行も可能。

③ サーバで実行する場合に必要なAWS環境
→ EC2上にCLIモードのKatalon環境を構築し、ヘッドレスブラウザと共に実行。
→ 有料版はKatalon TestOpsというクラウドプラットフォームからAWS連携実行も可能。
→ Docker化してECSやEKSに配置することも可能。

備考：
	•	GUIベースで操作しやすく、非エンジニアでも扱いやすい。
	•	無料版は機能制限あり。企業用途には有料版推奨。
	•	TestOpsと組み合わせれば、テスト管理やレポートも一元化できる。

⸻

■ TestComplete（有料）

① 主な機能
→ 商用GUI自動化プラットフォーム。Web、Windows、モバイルアプリに対応。
→ レコーディング、オブジェクト認識、スクリプト編集、データ駆動型テストなど高機能。

② クライアントで実行できるか
→ 実行可能。Windowsアプリとしてローカルにインストールして使用可能。
→ CLIモードやCIツールとの連携で自動実行も対応。

③ サーバで実行する場合に必要なAWS環境
→ WindowsベースのEC2インスタンスが必要。
→ 分散実行時はTestExecuteという実行エージェントを複数ノードに導入。
→ RDP（リモートデスクトップ）での保守・操作が前提。

備考：
	•	商用ライセンス制で導入コストは高め。
	•	機能は非常に強力で、大規模・安定運用の現場向け。
	•	スクリプト互換性が高く、開発者とテスターの協業に向いている。
